1. How the “Have a code?” button works

On every paywall in the ChessEver app, below the “Restore purchases” link, there is a “Have a code?” button. What happens when a user taps it depends on which device they are on:

iOS / iPad

Apple's official redemption window opens.

The user types the code into Apple's own UI. Apple validates it, charges the right amount (or marks it free), and starts the subscription. The ChessEver app instantly unlocks Premium when Apple confirms.

Android

A small ChessEver sheet asks for the code.

After they type it, the app opens the Play Store with that code already filled in. The user just taps Redeem. Coming back to ChessEver, Premium is unlocked automatically.

Important

Codes are generated by the stores (Apple and Google) — not by ChessEver. We do not manually create or enter codes anywhere in our backend. Your job is to use the store consoles to mint codes and share them.

2. iOS — App Store Connect

App Store Connect offers two kinds of codes. Pick the one that fits the use case:

TypeBest forHow users see itLimits
Promo Codes Press, reviewers, free giveaways, a few power users Free version of a specific subscription / IAP Up to 100 per quarter per app, expires in 28 days
Offer Codes Marketing campaigns, win-back, partner deals Discount, free trial, or pay-as-you-go offer Up to 10 active offers per subscription, millions of codes per offer

A. Generating Promo Codes (free copy of subscription)

  1. 1Sign in to appstoreconnect.apple.com with the ChessEver Apple ID.
  2. 2Click My Apps → ChessEver.
  3. 3In the left sidebar, open Features → Promo Codes.
  4. 4Click the + button next to the In-App Purchase you want to give away (e.g. ChessEver Premium Annual).
  5. 5Type how many codes you want (max 100 per quarter, shared across all IAPs).
  6. 6Agree to the terms → click Generate.
  7. 7Click Download Codes. Apple gives you a .txt file with one code per line.

Each code can only be used once. Codes expire 28 days after generation, whether redeemed or not.

B. Generating Offer Codes (discount or free trial)

  1. 1App Store Connect → My Apps → ChessEver → Subscriptions (under In-App Purchases & Subscriptions).
  2. 2Click the subscription product (e.g. Premium Annual).
  3. 3Scroll to Subscription Offers → click Create Offer Code.
  4. 4Fill out:
    • Reference name — internal label, e.g. Holiday2026.
    • Eligibility — New customers / Existing / Expired.
    • Offer type — Free, Pay as you go, or Pay up front.
    • Duration — e.g. 1 month free.
  5. 5Add the territories where the offer is valid.
  6. 6Choose how to deliver:
    • One-time-use codes — Apple generates a CSV with thousands of unique codes.
    • Custom code — One single shareable code (e.g. CHESS50) anyone can use, with a usage cap.
  7. 7Save → download the CSV (for one-time codes).

3. Android — Google Play Console

Google Play uses a single concept called Promotions. You can issue either one shareable code or a batch of unique codes.

Code styleBest forNotes
One-time codes Giveaways, individual gifting Up to 500,000 codes per promotion. Each code works once.
Custom (vanity) code Marketing campaigns, podcasts, partners One code, e.g. CHESS50. You set the redemption cap.

Generating a Promotion

  1. 1Sign in to play.google.com/console with the ChessEver Google account.
  2. 2Pick the ChessEver app.
  3. 3In the left sidebar: Monetize with Play → Promo codes. (In some accounts this is under Monetization setup → Promotions.)
  4. 4Click Create promo code.
  5. 5Fill out:
    • Promotion type — One-time codes or Custom code.
    • Promotion name — internal label.
    • Applicable products — pick the subscription or IAP (e.g. premium_annual).
    • Discount — Free or Percentage off.
    • Duration — for free trials, how many days/months are free.
    • Start & end date — when codes can be redeemed.
    • Number of codes (one-time) or max redemptions (custom).
  6. 6Click Create.
  7. 7Download the CSV (one-time) or copy the single code (custom).
Heads-up about Android subscriptions:

Google Play promo codes for subscriptions only support free trials — you cannot give a percentage discount on a subscription via a promo code. For one-time products you can do free or % off.

4. Sharing codes & testing

How to share codes with users

  • iOS direct link (recommended for marketing):
    https://apps.apple.com/redeem?ctx=offercodes&id=6752567269&code=<THECODE>
    Opens the App Store with the code pre-filled. One tap to redeem.
  • iOS in-app: User opens ChessEver → paywall → Have a code? → types it.
  • Android direct link: https://play.google.com/redeem?code=<THECODE>
    Opens Play Store with the code pre-filled.
  • Android in-app: User opens ChessEver → paywall → Have a code? → types it. The app launches Play Store automatically.
Example messages you can send

iOS user:

Tap to claim 1 month of ChessEver Premium on us:
https://apps.apple.com/redeem?ctx=offercodes&id=6752567269&code=ABC123XYZ

Android user:

Tap to claim 1 month of ChessEver Premium on us:
https://play.google.com/redeem?code=ABC123XYZ

Cross-platform / unknown device: share the raw code, e.g. “Open ChessEver, tap Have a code? on the upgrade screen, and enter ABC123XYZ.”

Testing before you ship codes to real users

  1. Generate one code first.
  2. On a real device, sign in with a different Apple ID / Google account than your developer account (sandbox / test accounts work).
  3. Open ChessEver → paywall → Have a code? → enter the code.
  4. Confirm Premium unlocks (the celebration animation plays and the paywall closes).
  5. Cancel/refund the test code in the store console so the batch isn't reduced.

5. Troubleshooting

ProblemLikely causeFix
“Code not valid” on iOS Code expired (28 days for promo codes), wrong region, or already redeemed Generate a fresh code; verify the user's country is in the offer's territory list
Play Store says “Promotion not available” Promotion not started yet, ended, or device country not allowed Check start/end dates and country of the Google account
Code redeemed but Premium not unlocked App didn't refresh entitlement Force-close and reopen ChessEver. The app re-checks RevenueCat on resume and unlocks automatically.
Same code redeemed twice One-time codes are single-use — second attempt fails Issue a new code; for repeat use, set up a custom/vanity code with a usage cap
User on iOS sees a blank sheet Older iOS versions or regional restriction Ask user to redeem via the direct App Store link instead
Quick mental model

Apple and Google handle billing, validation, and refunds. ChessEver only listens for the “user is now subscribed” signal via RevenueCat and flips Premium on. So if a code works in the store, it works in the app — no engineering ticket needed.

6. Partner-attributed vanity codes (optional)

Everything above gets you working codes. This section only matters if you plan to give an affiliate partner a single shareable code (e.g. JOHNCHESS) and you want commissions to flow to them when anyone redeems it — even users who never clicked their tracking link.

How it works (one paragraph)

The app stamps the typed code onto the user's RevenueCat record. The webhook checks three places, in order, to figure out which affiliate to credit:

  1. Did the user install via an affiliate's tracking link? → credit that affiliate.
  2. Did they install via a tracking link and redeem a code? → credit the install affiliate.
  3. Otherwise: does the typed code match a known affiliate code in our DB? → credit that affiliate.

Layer 3 is the new lever for partner vanity codes. To activate it for a given code, that exact string must exist in the affiliates table.

Setup checklist (per partner)

  1. 1Pick a code string. Short, uppercase, no spaces. Example: JOHNCHESS or PODCAST50.
  2. 2Create the code in the store(s).
    • Android (recommended): Play Console → Promo codes → Custom code with that exact string. Vanity attribution works fully on Android.
    • iOS: App Store Connect → Subscription Offer Codes → Custom code with the same string. Note: Apple's redemption sheet doesn't pass the typed code back to the app, so iOS attribution to a partner via vanity code only works if the user also installed via the partner's tracking link. For pure-organic iOS users redeeming a vanity code, you can still issue the code, but commission won't be auto-credited.
  3. 3Add a row to the affiliates table in Supabase (only if the partner doesn't already have one with that exact code). The engineering team can do this in a single SQL command, or you can use the Supabase dashboard:
    INSERT INTO affiliates (code, name, commission_rate, is_active)
    VALUES ('JOHNCHESS', 'John — Podcast partner', 0.30, true);
  4. 4Hand the code to the partner along with the direct-redeem URLs from §4.
  5. 5Watch the affiliate dashboard at aff.chessever.com. Conversions from this code will appear under that partner with status pending and clear after the refund window.
If you skip step 3, the code still works and unlocks Premium — but no partner gets credited. Use this when the code is internal-only (e.g. press, retention, win-back) and there is no commission to pay out.

What goes into the affiliates row

ColumnWhat to put
codeThe exact vanity code string. Must match the Play / ASC code character-for-character. Stored uppercase.
nameInternal label — partner name + campaign. e.g. “John — Podcast partner”.
commission_rateDecimal share, default 0.30 (= 30 %). Edit per deal.
is_activetrue. Set to false to retire the code (existing redemptions stay credited; new ones won't credit).
email / payout_detailsOptional. Fill if you'll be paying this partner.
When in doubt, skip it

For everyday giveaways (press kits, internal team, retention save-saves) you don't need any of §6. Just generate codes in App Store Connect / Play Console and share them. The DB only enters the picture when you decide to pay someone for sharing them.
